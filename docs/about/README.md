## 碎碎念

团队风格

- 实战派
- 学术派
- 工程派
- 算法派

设计模式：设计模式和类不在同一个层次，设计模式重要的是使用场景，更多的是在一定的基础上，去解决某些问题。

架构的原则，适度灵活，简单至上，越是简单的架构，遇到新的业务需求，改动和升级越容易。

学习新技能、掌握新知识的时候，应该用怎样的思路来进行？

- 理脉络，认场景，扣细节。

>商业分析的思维养成：第一部分是看数据, 第二部分是琢磨人性!
